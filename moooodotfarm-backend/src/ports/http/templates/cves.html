CVE-2026-00002 2026-05-01 Severity: HIGH Status: PATCHED
Click Fail: 100% reliable, every farm since 2026, pen escape link 47 bytes
Description

Most moooo.farm CVEs need an open pen gate or a herd-specific vector. Click Fail is a straight-line logic flaw - it needs neither. The same 47-charcter link roots every moooo.farm code distribution shipped since 2026.

The "you won" page shown to lucky visitors arriving at the farm with a Referrer header present placed the request's original path directly in an inline onclick handler as a JavaScript string literal. Although the templates HTML-escaped the value, HTML escaping is not sufficient to escape values in JavaScript strings. A crafted path containing an apostrophe was therefore able to terminate the JavaScript string literal and inject arbitrary JavaScript code into the page. A cunning attacker could set a trap for our visitors and lure them, unsuspecting, into the weeds.

Severity & Impact

Successful exploitation could result in arbitrary script execution on any subpage of moooo.farm except for the api documentation page. This could allow the attacker to perform a variety of extremely troublesome actions such as maliciously altering the names of admired cows or (what's worse) not letting the farm visitor admire the herd at all. Those actions could cause the cow in question or the farm visitor to experience troubling negative emotions. Experiencing such emotions is unacceptable in the context of moooo.farm.

Affected Versions

All versions from the introduction of the "you won" page up to the fix commit.

Patched Version

Patched in commit 68b481b2e0d71dbda0d161c0fe9b24439a72fde0.

Mitigation

The patch removed the inline onclick code. Click handlers are now attached via a script that reads the destination from each link's href property instead which is correctly escaped by the templating engine.

Exploitation Status

As of the date of this advisory, there is no evidence that this vulnerability was ever exploited in the wild or at the farm. The issue was discovered by an expert security researcher and disclosed in confidence, which caused the highly skilled and dedicated team behind moooo.farm to spring into action and ensure the safety of the herd and the farm visitors. Thanks to the star performance of everyone involved, the vulnerability can't be exploited.

CVE-2026-00001 2026-02-10 Severity: HIGH Status: PATCHED
Side Channel Attack on Shy Cows' Censored Names
Description

A side channel vulnerability in the handling of shy cows' names allowed an attacker to deduce the shy cow's name by submitting crafted names and analyzing the position of the shy cow in the herd. By observing how the position of a newly submitted cow related to other cows in the herd, the attacker could perform a binary search to reveal censored names of shy cows. This could violate the privacy expectations of shy cows in the herd.

Severity & Impact

Successful exploitation could result in the exposure of shy cows' names that were intended to remain private. No data corruption or denial of service is possible via this vector, but the privacy of the herd is at risk.

This vulnerability affects both the main https://moooo.farm/ web interface as well as the API endpoints available under https://moooo.farm/api.

Affected Versions

All versions before the fix commit and after the cow characters were introduced.

Patched Version

Patched in commit c23046b80b14eca192e1574b7a97fd2934208e2b.

Mitigation

The patch ensures that the sorting logic no longer leaks information about the shy cows' names via the cows' position in the list, making such side channel attacks infeasible. No action is required for existing cows or new cows.

Exploitation Status

As of the date of this advisory, there is no evidence that this vulnerability was ever exploited in the wild or at the farm. The issue was discovered internally during a routine code review and has only theoretical impact. No user-reported incidents or suspicious activity related to this vulnerability have been observed.
